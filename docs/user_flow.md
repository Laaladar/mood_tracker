# Пользовательские сценарии (User Flow)

Этот документ описывает логику взаимодействия пользователя с интерфейсом приложения Mood Tracker.

## 1. Сценарий: Вход и Регистрация

Диаграмма показывает процесс от момента захода на сайт до получения доступа к личному дневнику.

```mermaid
graph TD
    Start[Заход на сайт] --> Guest{Авторизован?}
    Guest -- Нет --> AuthPage[Экран входа/регистрации]
    Guest -- Да --> App[Экран дневника]

    AuthPage --> Login[Ввод логина и пароля]
    Login --> ServerCheck{Проверка на сервере}
    ServerCheck -- Ошибка --> Error[Сообщение об ошибке]
    Error --> AuthPage
    ServerCheck -- Успех --> App

    AuthPage --> Reg[Переключение на Регистрацию]
    Reg --> RegInput[Ввод данных: логин, ник, почта, пароль]
    RegInput --> RegCheck{Логин свободен?}
    RegCheck -- Нет --> RegError[Ошибка: логин занят]
    RegError --> Reg
    RegCheck -- Да --> App
```

## 2. Сценарий: Сохранение записи

Логика работы основной функции приложения после авторизации.

```mermaid
graph LR
    Input[Ввод настроения и текста] --> Click[Нажатие 'Сохранить']
    Click --> Validation{Поля заполнены?}
    Validation -- Нет --> Alert[Предупреждение]
    Validation -- Да --> API[Запрос POST /api/mood]
    API --> DB[Запись в SQLite]
    DB --> Refresh[Обновление списка истории на экране]
```

## 3. Сценарий: Выход из системы

```mermaid
graph TD
    A[Нажатие кнопки 'Выйти'] --> B[Запрос /api/logout]
    B --> C[Очистка сессии на сервере]
    C --> D[Перезагрузка страницы]
    D --> E[Возврат к экрану гостя]
```
