## Техническое задание (ТЗ): Приложение Mood Tracker (v1.0)## 1. Общие сведения
Mood Tracker — это персональный веб-дневник для отслеживания психоэмоционального состояния. Приложение позволяет пользователю регистрироваться, авторизовываться и вести хронологический лог своего настроения.
## 2. Стек технологий

* Backend: Python 3.x, Flask (фреймворк).
* Database: SQLite (реляционная база данных).
* Frontend: HTML5, CSS3 (Flexbox), JavaScript (Vanilla).
* Безопасность: Werkzeug (хеширование паролей).
* Инфраструктура: GitHub (репозиторий), Render (хостинг).

## 3. Функциональные требования## 3.1. Система авторизации и доступа

* Регистрация: Пользователь вводит логин (уникальный), никнейм, пароль и email. Система создает запись в БД, хеширует пароль и автоматически авторизует пользователя.
* Вход (Login): Проверка пары логин/пароль. При успехе создается серверная сессия (session).
* Выход (Logout): Удаление данных пользователя из сессии и возврат к гостевому экрану.
* Ограничение доступа: Анонимный пользователь видит только форму входа/регистрации. Доступ к дневнику открывается только после авторизации.

## 3.2. Управление записями (Дневник)

* Создание лога: Пользователь выбирает уровень настроения (1–10), пишет одно ключевое слово, описывает причину и фиксирует состояние счастья.
* Автоматизация: Система сама проставляет текущую дату и время (с указанием года) в момент сохранения.
* Персонализация: Пользователь видит в ленте только свои записи (фильтрация по user_id).
* Хронология: История записей отображается списком от новых к старым.

## 3.3. Пользовательский интерфейс (UI)

* Header: Фиксированная верхняя панель с логотипом слева и блоком управления аккаунтом (Приветствие/Выход или Вход/Регистрация) справа.
* Адаптивность: Оптимизация под мобильные устройства (сужение инпутов и шрифтов в шапке через медиа-запросы).
* Интерактивность: Цифровое отображение выбранного значения настроения рядом с ползунком.

## 4. Структура базы данных (Архитектура)## Таблица users (Пользователи)

* id: INTEGER (Primary Key)
* login: TEXT (Unique) — логин для входа.
* password_hash: TEXT — защищенный пароль.
* nickname: TEXT — имя, которое видит пользователь.
* email: TEXT — контактная почта.
* reg_date: TEXT — дата создания аккаунта.

## Таблица logs (Записи)

* id: INTEGER (Primary Key)
* user_id: INTEGER (Foreign Key) — связь с таблицей users.
* score: INTEGER — оценка настроения.
* word: TEXT — состояние одним словом.
* why: TEXT — подробный комментарий.
* happy: TEXT — отметка о состоянии счастья (Да/Нет).
* date: TEXT — дата и время записи.

## 5. Технические ограничения и безопасность

* Пароли хранятся только в виде необратимых хешей.
* База данных хранится в локальном файле mood_tracker.db на сервере.
* Все запросы к данным защищены проверкой активной сессии пользователя.
